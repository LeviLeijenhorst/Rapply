Privacy and Security Page Content Brief

Purpose
This document describes the content and structure a designer needs to build the privacy and security page for the website. It balances marketing clarity with legally accurate statements based on the attached Microsoft documentation.

Brand and Tone
- Calm, clear, and trustworthy.
- Use short sentences and plain language.
- Avoid legal claims that go beyond the facts in this document.
- Emphasize customer control and transparency.

Audience
- Prospective customers evaluating trust and compliance.
- Current customers needing clarity on data handling.
- Security and compliance reviewers.

Core Messaging
- We use Microsoft services in Europe.
- Real time speech to text does not store audio at rest in Microsoft systems.
- Batch transcription stores files only in customer specified storage, with customer controlled retention.
- Prompts and outputs for Microsoft Foundry models are processed as described by Microsoft and not used to train models.

Page Structure and Content

1. Hero Section
Title
Privacy and security you can trust

Subtitle
We use Microsoft services in Europe and follow clear, documented data handling practices.

Primary call to action
Talk to us about security

Secondary call to action
Download data processing addendum

2. Quick Facts
This is a short list with simple statements. Use icons.
- Microsoft regions in Europe.
- No Mistral.ai usage.
- No training on customer prompts or outputs.
- Customer controlled storage for batch transcription files.
- Data encrypted in transit.

3. What data we process
Use a two column layout with short explanations.

Audio input
Real time speech to text accepts audio that is processed in memory on Microsoft servers and not stored at rest.

Transcripts
Transcription text is generated from the audio input and returned to the customer.

Prompts and outputs
For Microsoft Foundry model usage, prompts and outputs are processed to provide the service and are not used to train Microsoft models or third party models.

4. How speech to text handles data
Use a simple diagram or step list. Keep it short.

Real time speech to text
- Audio is streamed to the service.
- Audio is processed in memory.
- Transcription is returned.
- No audio is stored at rest by Microsoft.

Batch transcription
- Audio input and output are stored in a customer chosen storage location.
- Retention is controlled by the customer, including time to live options.

Speaker separation
- Voice characteristics are used only to label speakers in the transcript.
- The voice characteristics are discarded after processing.

5. Data storage and location
Use a clear list.
- All Microsoft regions we use are within Europe.
- Customer data at rest is stored within the Microsoft region selected.
- Data in transit is encrypted.

6. Data use and training
Use a short paragraph.
Microsoft does not use prompts or outputs from Microsoft Foundry deployments to train Microsoft models, third party models, or any other models. Data is processed only to provide the service.

7. Customer control
Use a short list.
- You choose the storage location for batch transcription.
- You set retention for stored transcripts.
- You can delete data according to your own policies.

8. Security and compliance
Use a compact list and link to Microsoft resources.
- Microsoft follows ISO 27001, ISO 27002, and ISO 27018 practices.
- Access is controlled with least privilege and audited.
- Security incidents are reported to customers without undue delay.

9. Legal notes
Use smaller text and simple language.
- This page is for information only and is not legal advice.
- You are responsible for obtaining permissions for processing audio and content.
- For full legal terms, see the Microsoft Products and Services Data Protection Addendum.

Required links

Notes for the designer
- Use a clear hierarchy and plenty of spacing.
- Provide a simple data flow diagram for real time and batch transcription.
- Keep the page light and readable, with sections that can be scanned quickly.
